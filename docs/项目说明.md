# reTerminal E1002 A股看板 — 项目说明

> 本文档记录项目的实现原理、架构、关键决策，以及「下次想改东西时该改哪里」的指南。
> 读完能回答两个问题：**它怎么跑起来的？我想改某个功能去哪个文件？**

---

## 1. 这是什么

给 Seeed **reTerminal E1002**（7.3 英寸彩色电子纸，800×480，6 色）做的一个桌面看板，
**白天看 A 股、晚上看美股，按北京时间自动切换**：指数 + 自选股 + 规则化建议 + 天气 + 时钟。**自动刷新，不依赖电脑常开。**

电子纸的特点决定了它的用法：刷新慢（全屏刷一次十几秒）、只有 6 种颜色、但常亮不费电、不刺眼——所以它天生适合做「每隔几分钟更新一次的静态信息屏」，不适合做实时滚动/动画。

---

## 2. 整体架构（数据怎么从源头流到屏幕）

```
① 数据源         ② 抓取+打包          ③ 定时器          ④ 托管         ⑤ 显示
新浪行情  ─┐                        cron-job.org     GitHub Pages   reTerminal E1002
Open-Meteo ┘→ Python 脚本         (交易时段每15min   (公网固定网址)  (SenseCraft 每15min
            跑在 GitHub Actions     敲 GitHub API)                    拉一次页面刷新)
              ↓                        ↑                                 ↑
         生成 widget.html          绕过 GitHub 对新号              电子纸全屏刷新
         (数据+时钟内联)           定时任务的限流                  显示最新内容
```

**一句话**：云端定时生成一张最新页面发到公网，设备定时去拉这张页面刷新。全程你的电脑没参与。

---

## 3. 一次刷新的完整旅程（以盘中 10:23 为例）

1. **10:23** cron-job.org 到点 → 给 GitHub 发 POST 请求（带 token）喊"该更新了"
2. GitHub 启动一台临时服务器跑 Actions
3. `fetch_market.py` 抓新浪行情（上证+自选股）和 Open-Meteo 天气，算涨跌、信号、建议
4. `build_dist.py` 把数据+此刻北京时间**内联**进 HTML，生成 800×480 的 `widget.html`
5. 推送到 gh-pages 分支 → GitHub Pages 公网地址更新
6. 你的 E1002 到了它的 15 分钟刷新点 → 拉新页面 → 电子纸全屏刷新 → 显示最新股价

## 4. 各部分详解

### 4.1 显示层（页面设计）
- **固定画布 800×480**，禁响应式（墨水屏尺寸固定，响应式会让排版漂移）
- **三栏布局**：左=上证指数+走势线，中=自选股列表，右=天气+建议+状态
- **严格只用 6 色**（黑/白/红/黄/蓝/绿）。A 股「红涨绿跌」天然契合
- **视觉层级用字号/字重区分**，不用灰色——灰色在 6 色屏上会抖动成颗粒（这是初版模糊的根因）
- 走势线用 SVG 实色折线（不用 canvas/渐变）
- 文件：`index.html`（结构）、`styles.css`（6 色样式）、`app.js`（渲染逻辑）

### 4.2 数据层（行情/天气获取）
- **行情三级容错**：新浪（主）→ 东方财富（备）→ 保留上次数据（兜底）。任一源挂了不至于显示乱码
- 东财价格字段以「分」为单位，解析时 /100，并做合理性校验（防止字段映射错了输出离谱股价）
- 天气：Open-Meteo（免费、无需 key），WMO 天气码映射中文
- 分时走势点：东财 trends2 接口
- 全程 Python 标准库（urllib/json），零第三方依赖，CI 环境直接能跑
- 文件：`scripts/fetch_market.py`、`config/panel.json`（自选股/城市配置）
- **A 股 / 美股双时段**：北京时间 **22:00–次日 9:00** 显示美股（纳斯达克指数 + SPCX/AAPL/GOOG/TSLA/MSFT/AVGO/KO），其余时段显示 A 股。`fetch_market.py` 顶部 `is_us_window()` 按北京小时分流，两条路径输出**完全相同的 JSON 结构**，前端零改动即可切换。美股数据源：新浪 `gb_` 前缀（主）→ 东财 `105./106.` secid（备，`f43` 直接美元不除100）→ 保留上次（兜底）；中文名优先用 `config` 的 `us.names` 表。想改切换时段，调 `is_us_window()` 即可。

### 4.3 自动化流水线（核心：去电脑依赖）
这是整个项目最关键的工程部分。
- **GitHub Actions workflow**：checkout → fetch → build → 部署 gh-pages
- **定时器难题**：GitHub 对新账户的 scheduled workflow 严重限流（实测 13 小时仅触发 1 次）→ 用**外部定时器 cron-job.org**，按交易时段 cron 调用 `workflow_dispatch` API 绕过
- **海外 IP**：实测新浪在 GitHub 美国节点可用，无需切国内云
- **失败处理**：对比 `updatedAt`，fetch 没更新就跳过部署，保留上次成功数据（设备不会回退到初始样例）
- 文件：`.github/workflows/refresh.yml`、外部服务 cron-job.org

### 4.4 设备端（SenseCraft 显示）
- SenseCraft HMI 平台建 HTML Widget，填 Pages 的 widget.html 网址，刷新间隔设 15 分钟
- 设备是**拉取方**（定时去公网拉整页），不是推送方
- 设备端零代码、零状态，只负责"到点看公告栏"

## 5. 关键设计决策（为什么这么做）

| 问题 | 选择 | 原因 |
|---|---|---|
| 怎么显示？ | Web 页面，不写 ESP32 固件 | 渲染能力强、中文无障碍、改起来快，不碰嵌入式 |
| 在哪算/存？ | GitHub Actions + Pages | 免费、云端、自带 HTTPS，不用买服务器 |
| 谁来定时？ | cron-job.org（外部） | GitHub 对新号 cron 限流，外部定时器绕过 |
| 配色？ | 严格 6 纯色 | Spectra 6 只有 6 色，灰色/半透明会糊 |
| 时钟？ | 云端算好内联 | 设备不一定持续跑 JS，时间服务端算死 |
| 数据源？ | 新浪→东财→缓存 | 单源挂了不至于乱码 |

## 6. 修改指南（下次想改东西，去这里）

| 我想改… | 改哪里 | 改完要做什么 |
|---|---|---|
| 自选股 / 城市 | `config/panel.json` | push（自动触发一次更新） |
| 美股自选股 / 中文名 | `config/panel.json` 的 `us` 块 | push |
| A股/美股切换时段 | `scripts/fetch_market.py` 的 `is_us_window()` | push |
| 刷新频率 | cron-job.org 的 job 时间表 | 无需改代码 |
| 颜色 | `styles.css` 的 `:root` 变量（只能在这 6 色内调） | push，本地可先 `python3 scripts/build_dist.py` 预览 |
| 页面布局/文字 | `index.html` + `styles.css` + `app.js` | push |
| 分析建议的规则 | `app.js` 的 `buildAdvice()` / `signalFor()` | push |
| 数据源 | `scripts/fetch_market.py` | push |
| token（1年到期） | GitHub 重新生成 PAT → cron-job.org 更新 Authorization 头 | 无需改代码 |
| 设备刷新间隔 | SenseCraft 平台该页面设置 | 无需改代码 |

**本地预览改动**：
```bash
python3 scripts/build_dist.py
python3 -m http.server 4173
# 浏览器开 http://localhost:4173/dist/widget.html
```

## 7. 排错指南（常见问题）

| 现象 | 排查方向 |
|---|---|
| 股价不刷新 | ① cron-job.org 最近有没有成功执行（看它日志 HTTP 状态）② GitHub Actions 最近 run 有没有 success（github.com/.../actions）③ fetch 是否拿到数据（看 run 日志的 `Source:` 行） |
| 显示模糊/有颗粒 | 检查 `styles.css` 是否混入了灰色/半透明/渐变（应只有 6 个 hex 色） |
| 数据全是 -- 或乱码 | 数据源全挂了，看 fetch 日志；检查网络/接口是否变更 |
| Pages 打不开 | GitHub 仓库 Settings → Pages 确认 Source=gh-pages；等 CDN 1-2 分钟 |
| GitHub cron 不触发 | 正常现象（新号限流），靠 cron-job.org 兜，不用管 GitHub 自己的 schedule |

## 9. 通俗版原理（给非技术的人讲）

把它想象成一个「会自己更新股价的电子告示牌」：

1. 网上有免费的地方能取到最新股价（新浪）
2. 我在 GitHub（免费代码网站）上存了程序，GitHub 会**到点自动**：取股价 → 拼成一张新画面 → 发到一个固定网址
3. 因为 GitHub 自己的闹钟对新号不准时，我**另请了一个准时的网站（cron-job.org）**到点去催它
4. 我的墨水屏**每 15 分钟自己去那个网址看一眼**最新画面，然后刷新显示

整个过程在云端转，**电脑关了也照跑**。屏幕只是个定时看公告的角色。
